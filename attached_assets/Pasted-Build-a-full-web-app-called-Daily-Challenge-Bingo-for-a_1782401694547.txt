Build a full web app called Daily Challenge Bingo for a church/youth spiritual growth activity.
This is not a simple landing page. It must be a real web application with an admin panel, participant accounts, personalized bingo cards, hidden challenge boxes, and daily reflections.

Tech Stack

Use:

- Flask (Python backend)
- SQLite database
- Bootstrap 5 for UI styling
- Clean HTML/CSS/JavaScript
- Organize the project properly so it can be edited later

Main Concept

The app is for a group activity where each participant gets a personalized Bingo card with daily challenges.
Each bingo box starts hidden and is revealed when the participant clicks it.
At the end of the day, the participant submits a short reflection.

The theme is Christian self-growth / daily spiritual challenge, but the app should look modern, clean, and mature — not childish.

---

User Roles

The app must have 2 roles:

1) Admin

The admin is the teacher/leader (Paula or another leader).
Admin must have a secure login.

2) Participant

Participants log in and see their own personalized bingo card only.

---

Core Features

A) Authentication

Admin login

- Admin can log in securely.

Participant login

- Each participant can log in with username + password.
- After login, participants are redirected to their own dashboard.

---

B) Admin Panel

Create a proper Admin Dashboard with a clean sidebar or top navigation.

Admin must be able to:

1. Create participants
   
   - name
   - username
   - password
   - optional email

2. View all participants

3. Create a personalized bingo card for each participant
   
   - Each card should be 3x3 for now
   - Each box contains:
     - challenge text
     - category
   - Categories can be one of:
     - Spirit
     - Mind
     - Body
     - Health
     - People

4. Edit participant bingo cards
   
   - Update challenge text
   - Change category
   - Reset a box if needed

5. View participant progress
   For each participant, admin should be able to see:
   
   - which boxes are still hidden
   - which boxes have been revealed
   - which boxes have been marked done
   - timestamps if possible

6. View daily reflections
   Admin should be able to see all reflections submitted by participants.

---

C) Participant Dashboard

Each participant must have a dashboard that shows:

1. Welcome section

- Participant name
- Short heading like: “Your Daily Challenge Bingo”

2. Personalized Bingo Card

Display a 3x3 Bingo grid.

Each box must have 3 states:

State 1: Hidden

- The box should look closed/covered.
- Show a button or label like:
  - “Reveal”
  - or “Open Challenge”
- The challenge text should NOT be visible yet.

State 2: Revealed

- When participant clicks the box, the challenge is revealed.
- The box now shows:
  - challenge text
  - category badge
- Save that state in the database.

State 3: Completed

- Participant can click Mark as Done
- Completed boxes should have a visual done state.

Important:

Participants should only be able to reveal and complete their own boxes.

---

D) Daily Reflection

Below the bingo card, add an End of Day Reflection section.

Participant should submit:

1. What I chose today
2. What I actually did
3. What changed / what impact did it have?

Save reflections in the database with:

- participant id
- date
- reflection text fields

Admin must be able to review these later.
